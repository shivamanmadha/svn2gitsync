Following section describes  'Reasons to go with

Reasons to go with CVS/SVN  - 
  CVS and SVN are the types of Versioning systems. 
These two work in 
 different ways. The following URL lists the 
differences between these two. 
http://www.pushok.com/soft_svn_vscvs.php 
Some of the advantages of using Subversion are listed 
below. 
    * Using SVN we can revert to older versions of a 
file, directory, or an 
entire website. 
    * Helps in case your hard drive dies 
    * Helps in case you changed something and wished 
you hadn't 
    * Helps in case you coded something and didn't 
realize it broke something 
else until later. 
    * Helps in case you are working in a project with 
one or more other 
people and needed a way to keep your files in sync 
    * Branching releases. Concurrently develop an 
unstable development 
 version as well as fix bugs in the stable production 
version. 
    * Multiple developers can work on the same 
catalog 
and even the same file 
at the same time. 
    * SVN is better than ftp for file transfer, 
because 
it automatically 
downloads only changed files, and even then, only the 
portion of the file 
that has changed (using patches). 
    * SVN can automatically merge two simultaneous 
writes to the same file by 
different developers. 
    * Allows one to keep track of the changes that 
have 
been made over time 
(many release managers repackage SVN commit logs into 
WHATSNEW, HISTORY, 
and/or NEWS files). 
   * Acts as a Central Repository bcaz of this taking 
backups would be easy, 
Administrator need not to take the backup from 
individual users. 
   Note : For more information you can also find many 
tutorial by just typing 
words like 'advantages CVS' (OR) 'Uses CVS'  (OR) 
'introduction cvs' etc.. in 
 google.
